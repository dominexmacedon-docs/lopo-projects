# Contributing to lopo-projects

First off, thanks for checking out the project! Whether you're fixing a typo, optimizing a routing pattern, or adding a completely new Lopo app example, your help is awesome.

Here is a straightforward guide on how to get your ideas merged.

---

## How to Help Out

There are a few ways you can contribute to this repository:

* **Fix Bugs or Typos:** Notice a broken route or a bad HTML tag? Open a quick pull request to fix it.
* **Add New Projects:** If you've built a practical web feature or microservice using Lopo, we’d love to include it as a reference example.
* **Improve Documentation:** Help make our README files or project comments clearer for developers trying to learn the language.

---

## Guidelines for New Examples

If you are adding a new project folder, please make sure it fits the vibe of the repo:

1. **Keep it Clean:** Separate your backend logic (`.lopo`) from your visual presentation (`.html`).
2. **Stick to Lopo Paradigms:** Ensure your scripts consistently use explicit `define` declarations and structured blocks.
3. **Real-World Focus:** We want examples that show how to build actual features (like forms, API endpoints, or middleware), not just abstract code snippets.
4. **Self-Contained:** Keep each project in its own self-contained directory so someone can jump right in and run it.

---

## Step-by-Step Process

1. **Fork the Repo:** Grab your own copy of the repository to work on.
2. **Create a Branch:** Give your branch a simple, descriptive name (e.g., `feature/add-login-example`).
3. **Test Your Code:** Fire up your local Lopo runtime and make sure the server boots, the routes work, and the templates render correctly in the browser.
4. **Submit a PR:** Open a Pull Request back to this repository. Give a brief summary of what your example does or what bug you fixed.

---

## Ground Rules

Be respectful and constructive when reviewing other people's code or discussing changes. Check out our **Code of Conduct** for the community standards we all follow here.
