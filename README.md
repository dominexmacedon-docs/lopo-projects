# lopo-projects

This is a personal playground for building actual web stuff with **LopoLang**. Instead of dry documentation or generic "Hello World" examples, everything here is built around real app features - showing how Lopo's backend, data flow, and native template engine work together when building for the web.

---

## What's in Here?

These projects focus on the day-to-day pieces of web development:

* **Handling Data:** Grabbing URL parameters, parsing query strings, and capturing incoming form submissions.
* **Dynamic Views:** Using Lopo's built-in rendering engine to pass backend data directly into clean HTML templates without relying on heavy external libraries.
* **Real-World Scenarios:** Setting up custom logging middleware, handling 404 pages, and toggling maintenance modes on the fly.

---

## The Approach

Every project follows a few simple design choices:

* **Clear Structure:** Keeping code highly readable by using Lopo's explicit variable declarations and structured blocks.
* **Separation of Concerns:** Backend logic and HTML visuals are kept completely separate. The scripts handle the heavy data lifting, and the templates just handle the display.

---

## Running Things

To check out any of the examples:

1. **Get the files:** Grab a copy of the repository.
2. **Start the server:** Run the project script through your local Lopo interpreter from your terminal.
3. **See it live:** Pop open your browser and head to the local port printed in your console to interact with the rendered pages.
